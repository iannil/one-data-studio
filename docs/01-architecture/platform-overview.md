# 平台概览

## 平台定位

将 **Alldata**（数据治理与开发）、**Cube Studio**（云原生一站式机器学习/MLOps平台）和 **Bisheng**（大模型应用开发平台/LLMOps）这三个平台结合在一起，构建出一个**全链路、端到端的企业级"数据+AI"智能融合平台**。

这个平台打通了从**原始数据治理**到**模型训练部署**，再到**生成式AI应用构建**的完整价值链。

我们可以将这个融合平台定义为：**企业级 DataOps + MLOps + LLMOps 全栈智能基础设施**。

## 核心价值主张

### 打破数据与AI的孤岛

通常数据团队（用 Alldata）和算法团队（用 Cube Studio）是分开的。结合后，**Feature Store（特征平台）** 更加顺滑，算法工程师可以直接使用治理好的高质量数据，无需重复造轮子清洗数据。

### 结构化与非结构化的统一

Alldata 擅长结构化数据，Bisheng 擅长非结构化文档解析。结合后，企业可以构建**"ChatBI"**——既能查文档里的知识，又能直接用自然语言查询数据库里的销售报表（Text-to-SQL）。

### 私有化大模型落地的完整闭环

很多企业只想用 Bisheng 做应用，但缺乏模型微调能力；或者只有 Cube Studio 炼了模型，但缺乏好用的应用构建工具。

**三者结合 = 私有数据 (Alldata) + 私有算力/模型 (Cube Studio) + 私有应用 (Bisheng)**。这构成了最安全的企业级 AGI 解决方案。

### 全生命周期治理

从数据血缘（Alldata）到模型血缘（Cube Studio）再到应用日志，整个链路可追溯。如果 AI 回答错了，可以一路追溯是 Prompt 问题，模型过拟合问题，还是原始数据本身就是脏数据。

## 平台比喻

将这个平台想象成一个精密的工厂，分为三个核心车间：

* **底层基础（Alldata）：数据底座与供应链**
  * **角色**：负责"原材料"的采集、清洗和标准化。
  * **功能**：提供 DataOps 能力。它负责连接各种数据库，进行 ETL 开发、数据治理、元数据管理和数据质量监控。

* **中层核心（Cube Studio）：模型工厂与计算引擎**
  * **角色**：负责"加工与制造"。
  * **功能**：提供 MLOps 能力。利用 Alldata 处理好的数据，进行传统机器学习模型的训练或大模型（LLM）的微调。

* **上层应用（Bisheng）：业务装配与智能交互**
  * **角色**：负责"产品包装与分发"。
  * **功能**：提供 LLMOps 能力。它利用 Cube Studio 部署的模型服务，结合 Alldata 提供的知识库数据，通过 RAG 和 Agent 编排，快速搭建面向最终用户的应用。

## 总结

将 Alldata、Cube Studio、Bisheng 结合，得到的不仅仅是一个工具箱，而是一个**企业数字化转型的操作系统**。

* **Alldata 是"地基"**（管好数据）。
* **Cube Studio 是"引擎"**（造好模型）。
* **Bisheng 是"界面"**（用好AI）。

它让企业具备了从原始数据中提炼智能，并迅速将其转化为生产力的完整能力。
